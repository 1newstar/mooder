从源码安装#

除了使用docker一键安装以外，Mooder还支持从源码安装。这一章我介绍一下如何从源码安装Mooder，并配置数据库。

首先，Mooder是基于Python3的（不兼容Python2.7），请先安装Python 3。

1. 安装必要依赖#

安装必要依赖：

apt-get update && apt-get install libpq-dev libjpeg-dev zlib1g-dev libfreetype6-dev

拉取代码，进入源码目录。

安装python依赖：

pip3 install -r requirements.txt

2. 修改配置文件#

Mooder包含一个基础配置文件（mooder/settings.py）和一个生产环境配置文件(mooder/settings_production.py)。二者是继承关系，settings_production.py继承所有settings.py定义的配置，并通过定义相同名字的变量来修改配置。

如，默认settings.py中定义DEBUG=True，即开启调试模式；而settings_production.py中定义DEBUG=False，就覆盖了settings.py的设置，关闭了Django的调试模式。

通常情况下，我们只需要修改settings_production.py中的选项即可完成对Mooder的配置。如果你需要更多自定义的配置，请阅读后面的文档。

settings_production.py默认如下：

from mooder.settings import *

DEBUG = False
ALLOWED_HOSTS = ['*']
SECURE_CONTENT_TYPE_NOSNIFF = True
SECURE_BROWSER_XSS_FILTER = True
X_FRAME_OPTIONS = 'DENY'

DATABASES = {
    'default': {
        'ENGINE': os.environ.get('DB_ENGINE', 'django.db.backends.postgresql') ,
        'NAME': os.environ.get('DB_NAME'),
        'USER': os.environ.get('DB_USER'),
        'PASSWORD': os.environ.get('DB_PASSWORD'),
        'HOST': os.environ.get('DB_HOST'),
        'PORT': os.environ.get('DB_PORT'),
    }
}

EMAIL_HOST_USER = os.environ.get('EMAIL_HOST_USER')
EMAIL_BACKEND = os.environ.get('EMAIL_BACKEND', 'django_mailgun.MailgunBackend')
if EMAIL_BACKEND=='django_mailgun.MailgunBackend':
    MAILGUN_ACCESS_KEY = os.environ.get('MAILGUN_ACCESS_KEY')
    MAILGUN_SERVER_NAME = os.environ.get('MAILGUN_SERVER_NAME')

if EMAIL_BACKEND == 'django.core.mail.backends.smtp.EmailBackend':
    DEFAULT_FROM_EMAIL = os.environ.get('DEFAULT_FROM_EMAIL')
    EMAIL_HOST = os.environ.get('EMAIL_HOST')
    EMAIL_PORT = os.environ.get('EMAIL_PORT')
    EMAIL_HOST_USER = os.environ.get('EMAIL_HOST_USER')
    EMAIL_HOST_PASSWORD = os.environ.get('EMAIL_HOST_PASSWORD')
    EMAIL_USE_SSL = os.environ.get('EMAIL_USE_SSL', None)
    if EMAIL_USE_SSL and EMAIL_USE_SSL.lower() == 'false':
        EMAIL_USE_SSL = None
    EMAIL_USE_TLS = os.environ.get('EMAIL_USE_TLS', None)
    if EMAIL_USE_TLS and EMAIL_USE_TLS.lower() == 'false':
        EMAIL_USE_TLS = None

STATIC_URL = '/static/'
STATIC_ROOT = os.path.join(BASE_DIR, 'mooder', 'static_cdn')

MEDIA_URL = '/media/'
MEDIA_ROOT = '/data'

第一行说明了settings_production.py与settings.py的继承关系。后面开始是覆盖默认项的配置项。

其中有几个选项必须配置：

  • DEBUG 调试模式，生产环境下务必关闭调试模式。
  • ALLOWED_HOSTS 允许哪些Host访问，默认为*，不限制。
  • DATABASES 数据库配置。
  • MEDIA_ROOT 数据文件（附件、头像等）存放地址，一定要有写权限。

关于Email的相关配置，见邮箱配置一节。

其他未提到的选项，可以不用修改。

3. 数据库配置#

Mooder默认使用postgres数据库，但其也可以使用Django支持的任何数据库引擎，具体配置方法见 Django文档

example 1. postgres配置：

DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.postgresql',
        'NAME': 'mydatabase',
        'USER': 'mydatabaseuser',
        'PASSWORD': 'mypassword',
        'HOST': '127.0.0.1',
        'PORT': '5432',
    }
}

example 2. mysql配置

DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.mysql',
        'NAME': 'test',
        'USER': 'root',
        'PASSWORD': 'root',
        'HOST': '127.0.0.1',
        'PORT': '3306',
    }
}

example 3. sqlite配置

DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.sqlite3',
        'NAME': os.path.join(BASE_DIR, 'db.sqlite3'),
    }
}

注意，使用mysql或postgres前需要安装一些必要的库。例如，使用postgres需要安装如下库：

sudo apt-get install postgresql-client
pip install psycopg2

Mysql需要安装如下库：

sudo apt-get install mysql-client
pip install mysqlclient

更多支持与配置（如数据库字符）还是参考Django的文档。

4. 生成加密密钥#

在settings.py中有一个SECRET_KEY（加密密钥），你可以直接给其赋予一个随机值：

SECRET_KEY = 'any random string'

默认情况下，Mooder会从项目根目录读取一个文件“.secretkey”，将其内容作为项目的密钥。所以，我们可以执行以下命令生成一个64位长度的密钥文件：

sudo apt-get install openssl
openssl rand -out .secretkey 64

5. 指定配置文件#

Mooder启动的时候，会默认加载mooder/settings.py作为配置文件。为了使用自定义配置文件，我们需要设置一个环境变量，指定自定义配置文件的位置：

# DJANGO_SETTINGS_MODULE 环境变量的值是Python包格式
export DJANGO_SETTINGS_MODULE=mooder.settings_custom

# 运行Mooder
python3 manage.py runserver

Windows下使用set来指定环境变量：

set DJANGO_SETTINGS_MODULE=mooder.settings_custom
py -3 manage.py runserver

如果你使用系统自带的生产环境配置文件（mooder/settings_production.py），也需要定义该环境变量。

在指定了环境变量DJANGO_SETTINGS_MODULE后，Mooder就会使用DJANGO_SETTINGS_MODULE指向的配置文件。

一定要先指定这个环境变量，在进行后面的6、7等操作，否则你定义的配置文件就不会生效。

6. 创建初始化管理员账号#

详细请阅读 管理员与审核员 一章。

简单来说，创建管理员只需执行如下命令：

python3 manage.py createsuperuser

然后根据提示，填写昵称、邮箱、密码即可。

7. 启动Mooder#

首先初始化数据库：

python3 ./manage.py migrate

生成静态文件：

python3 ./manage.py collectstatic

运行：

python3 ./manage.py runserver

以上命令会将mooder运行在默认端口（8000）。

生产环境下不建议直接使用Django自带的Webserver，请使用nginx + gunicorn或Apache mod-uwsgi部署。具体部署方法参考 Django文档。
