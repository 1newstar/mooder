邮箱配置#

注册Mailgun#

如果你选择使用Mailgun作为邮件客户端，就需要注册一个Mailgun账号( http://www.mailgun.com/ ) 。否则请跳过这一节。

注册Mailgun，并绑定自己的域名作为邮件发送方(或使用官方提供的一个域名邮箱)。在域名详情页面找到自己的API BASE URL和API KEY，后面会用到。

从Docker安装（包括daocloud）请阅读这一节#

从Docker安装的Mooder，许多配置会动态更改，所以将其放在环境变量中，便于用户在docker容器外对配置进行修改。

如果使用Mailgun，则需要填写以下环境变量：

  • EMAIL_BACKEND 值为 django_mailgun.MailgunBackend，说明使用了Mailgun
  • MAILGUN_SERVER_NAME 刚才在MAILGUN里找到的API Base URL，只用填写https://api.mailgun.net/v3/后面的部分，如mail.leavesongs.com
  • MAILGUN_ACCESS_KEY 刚才在MAILGUN里找到的API Key
  • EMAIL_HOST_USER 发邮件时默认使用的邮箱，比如 no-reply@mail.leavesongs.com ，这个可以自己定义，但后缀一定是你绑定Mailgun的域名
  • DEFAULT_FROM_EMAIL 发送邮件的邮箱

如果使用SMTP，则填写以下环境变量

  • EMAIL_BACKEND 值为 django.core.mail.backends.smtp.EmailBackend，说明使用SMTP做发信方式
  • EMAIL_HOST SMTP服务主机地址
  • EMAIL_PORT SMTP服务端口
  • EMAIL_HOST_USER SMTP登录用户名，也是后面邮件时发件人的邮箱
  • EMAIL_HOST_PASSWORD SMTP登录密码
  • EMAIL_USE_SSL SMTP发送是否使用SSL，设置为False或删除这个环境变量，则表示不使用SSL
  • EMAIL_USE_TLS SMTP发送是否使用TLS，同SSL
  • DEFAULT_FROM_EMAIL 发送邮件的邮箱

从源码安装请阅读这一节#

从源码安装需要更改配置文件，邮箱的相关设置也会在配置文件中进行修改。首先阅读这一章，了解Mooder的配置文件如何工作。

然后，配置文件settings_production.py中默认使用了如下选项：

EMAIL_HOST_USER = os.environ.get('EMAIL_HOST_USER')
EMAIL_BACKEND = os.environ.get('EMAIL_BACKEND', 'django_mailgun.MailgunBackend')
DEFAULT_FROM_EMAIL = os.environ.get('DEFAULT_FROM_EMAIL', EMAIL_HOST_USER)
if EMAIL_BACKEND=='django_mailgun.MailgunBackend':
    MAILGUN_ACCESS_KEY = os.environ.get('MAILGUN_ACCESS_KEY')
    MAILGUN_SERVER_NAME = os.environ.get('MAILGUN_SERVER_NAME')

if EMAIL_BACKEND == 'django.core.mail.backends.smtp.EmailBackend':
    EMAIL_HOST = os.environ.get('EMAIL_HOST')
    EMAIL_PORT = os.environ.get('EMAIL_PORT')
    EMAIL_HOST_USER = os.environ.get('EMAIL_HOST_USER')
    EMAIL_HOST_PASSWORD = os.environ.get('EMAIL_HOST_PASSWORD')
    EMAIL_USE_SSL = os.environ.get('EMAIL_USE_SSL', None)
    if EMAIL_USE_SSL and EMAIL_USE_SSL.lower() == 'false':
        EMAIL_USE_SSL = None
    EMAIL_USE_TLS = os.environ.get('EMAIL_USE_TLS', None)
    if EMAIL_USE_TLS and EMAIL_USE_TLS.lower() == 'false':
        EMAIL_USE_TLS = None

相关说明如下：

  • EMAIL_BACKEND 使用哪种邮件客户端，可供选择的有SMTP方式django.core.mail.backends.smtp.EmailBackend、Mailgun方式django_mailgun.MailgunBackend。建议使用SMTP。
  • DEFAULT_FROM_EMAIL 发送邮件的发件人，默认与SMTP登录的邮箱相同

如果你使用SMTP方式，请配置如下一些选项：

  • EMAIL_HOST SMTP服务地址
  • EMAIL_PORT SMTP服务端口
  • EMAIL_HOST_USER 登录SMTP服务的邮箱。
  • EMAIL_HOST_PASSWORD 邮箱密码
  • EMAIL_USE_SSL 是否使用SSL
  • EMAIL_USE_TLS 是否使用TLS

如果你使用Mailgun方式，配置如下选项：

  • MAILGUN_ACCESS_KEY 刚才在MAILGUN里找到的API Key
  • MAILGUN_SERVER_NAME 刚才在MAILGUN里找到的API Base URL，只用填写https://api.mailgun.net/v3/后面的部分，如mail.leavesongs.com
  • EMAIL_HOST_USER 登录SMTP服务的邮箱（也是后面发送邮件的邮箱）。

修改完成后，你的配置类似这样：

EMAIL_HOST_USER = 'no-reply@example.com'
EMAIL_BACKEND = 'django.core.mail.backends.smtp.EmailBackend'
DEFAULT_FROM_EMAIL = EMAIL_HOST_USER
EMAIL_HOST = 'smtp.example.com'
EMAIL_PORT = 25
EMAIL_HOST_PASSWORD = 'password'
EMAIL_USE_SSL = False
EMAIL_USE_TLS = False
