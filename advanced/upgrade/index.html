更新与升级#

Mooder会持续发布更新，使用docker-compose可以一键更新：

# 拉取最新代码
git pull 

# 重新编译Mooder镜像
sudo docker-compose build

# 重新启动容器
sudo docker-compose up -d

所以，强烈建议使用docker-compose在生产环境部署Mooder。

当然，使用源码安装Mooder的同学更新也很方便，只需重新拉取代码、更新数据库与静态文件、并重启Web容器：

git pull
python3 manage.py migrate
sudo python3 manage.py collectstatic
sudo service apache restart

但该方法可能会存在其他问题。比如，源码中增加了其他依赖，你就需要重新进行Python依赖的安装：

pip3 install -r requirements.txt

更新过程中有任何问题，可以开issue告诉我。
